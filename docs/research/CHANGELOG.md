# Objectif

Ce document retrace l'ensemble des évolutions du projet.

Contrairement au changelog classique d'un logiciel, celui-ci documente :

- les avancées scientifiques ;
- les évolutions de l'architecture ;
- les nouvelles idées ;
- les résultats expérimentaux ;
- les décisions importantes ;
- les changements de stratégie.

Il constitue la mémoire chronologique du laboratoire.

---

# Philosophie

Chaque modification importante doit être enregistrée.

Une évolution importante peut concerner :

- le code ;
- la documentation ;
- les données ;
- les modèles ;
- les expériences ;
- les résultats ;
- les décisions ;
- les processus.

# Politique de versionnement

Le projet utilise Semantic Versioning.

```
MAJOR.MINOR.PATCH
```

---

## MAJOR

Évolution majeure.

Exemple :

- changement complet d'architecture ;
- nouveau pipeline.

---

## MINOR

Nouvelle fonctionnalité.

Exemple :

- ajout d'un modèle ;
- ajout d'une nouvelle stratégie.

---

## PATCH

Correction.

Exemple :

- bug ;
- documentation ;
- optimisation.
